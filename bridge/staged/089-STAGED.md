## Objective

Build the production Wormhole MCP server. This is the foundational Sprint 3 infrastructure — Kira's drain and all other automated agents depend on it. The server runs as a native macOS process, registered in Claude Desktop, and exposes four tools that Cowork agents call instead of writing through VirtioFS.

## Context

Full ADR: `docs/architecture/WORMHOLE-ADR.md`. Read it before touching any code.

Summary: Cowork runs in a Linux VM. File writes from inside the VM cross the VirtioFS boundary and trigger macOS permission prompts per write. Automated pipelines stall on these prompts. The fix: a native macOS MCP server (node process) that accepts write tool calls via stdio and performs the writes directly on macOS — no sandbox boundary crossed, no prompt.

POC lives at `wormhole-poc/server.js`. It is registered in `claude_desktop_config.json` and confirmed working (see ADR §Evidence). You may read it for reference but do NOT modify or delete it yet — that is slice 091 (cutover).

Security rules from the ADR are non-negotiable — enforce them in code, not by convention:
1. Reject absolute paths
2. Reject `..` segments
3. Canonicalize resolved path, assert it starts with `WORKSPACE_ROOT`
4. Never read env vars, secrets, or files outside `WORKSPACE_ROOT`

Stdio protocol rule: any `console.log` corrupts the MCP protocol stream. ALL logging goes to stderr only.

## Tasks

1. Create `wormhole/` directory at the repo root.

2. Create `wormhole/package.json`:
   - Name: `@ds9/wormhole`
   - Main: `server.js`
   - Type: `commonjs`
   - Engines: node `>=20`
   - Dependencies: `@modelcontextprotocol/sdk` (same version as used in wormhole-poc — check `wormhole-poc/package.json`)
   - Scripts: `start: node server.js`

3. Create `wormhole/config.js`:
   - Reads `WORKSPACE_ROOT` from `process.env.WORMHOLE_WORKSPACE_ROOT`
   - Fails fast (exits 1 with stderr message) if unset or if the path does not exist as a directory
   - Exports `{ WORKSPACE_ROOT }` as an absolute resolved path

4. Create `wormhole/logging.js`:
   - Exports `log(level, message, meta)` — writes structured JSON to **stderr only**
   - Levels: `info`, `warn`, `error`
   - Format: `{ ts, level, message, ...meta }`

5. Create `wormhole/security.js`:
   - Exports `safePath(relativePath, workspaceRoot)` — resolves and validates a relative path
   - Throws a descriptive error (not silent) if: path is absolute, contains `..`, or resolves outside `workspaceRoot` after canonicalization
   - Returns the resolved absolute path on success

6. Create `wormhole/server.js` — the MCP server:
   - Uses `@modelcontextprotocol/sdk` stdio transport
   - Registers four tools:

   **`wormhole_write_file(path, content, options?)`**
   - `options.encoding` defaults to `utf-8`
   - Creates parent directories if they don't exist (`fs.mkdirSync(..., { recursive: true })`)
   - Overwrites or creates the file

   **`wormhole_append_jsonl(path, line)`**
   - `line` must be a valid JSON object (parse-validate before appending)
   - Appends `JSON.stringify(line) + '\n'` to the file
   - Creates parent directories if missing

   **`wormhole_move(from, to)`**
   - Both `from` and `to` validated through `safePath`
   - Uses `fs.renameSync`

   **`wormhole_delete(path)`**
   - Single file only — no recursive directory delete
   - Validated through `safePath`
   - Uses `fs.unlinkSync`

   **`wormhole_ping(nonce)`**
   - `nonce` is a short string generated by the caller (Cowork agent)
   - Writes `bridge/wormhole-ping.json` with `{ nonce, ts }` to the workspace root
   - No path validation needed — path is hardcoded to `bridge/wormhole-ping.json`
   - Returns `{ ok: true, nonce, ts }`
   - Purpose: bidirectional health check. Caller writes nonce via Wormhole (tests write path), then reads `bridge/wormhole-ping.json` back via VirtioFS and confirms nonce matches (tests read path). Also serves as consent preflight — first call per session triggers the one-time MCP consent click in Claude Desktop.

   - Every tool handler wraps its body in try/catch and returns a structured error on failure (never throws to the SDK)
   - All logging to stderr via `logging.js`

7. **Write a health sidecar on every successful tool call.** After each tool handler succeeds (before returning the response), append to `bridge/wormhole-heartbeat.json` (overwrite, not append — only last call matters):
   ```json
   { "ts": "<ISO 8601>", "tool": "<tool_name>", "path": "<relative_path>" }
   ```
   This write uses `fs.writeFileSync` directly (the Wormhole server is already on macOS — no prompt). Wrap in try/catch; failure must not affect the tool response.

   Add `bridge/wormhole-heartbeat.json` to `.gitignore` in the repo.

8. Write integration tests at `wormhole/tests/security.test.js`:
   - Path traversal attempt (`../../../etc/passwd`) → rejected
   - Symlink escape → rejected
   - Absolute path → rejected
   - Valid relative path → resolves correctly
   - `wormhole_append_jsonl` with invalid JSON → returns error, does not write
   - Write to missing parent dir → parent created, write succeeds

   Use Node's built-in `assert` module (no test framework dependency).

9. Run `npm install` in `wormhole/` and run the tests: `node wormhole/tests/security.test.js`

10. Commit on branch `slice/089-wormhole-core`:
   ```
   feat(089): Wormhole MCP server core — permissionless macOS writes
   ```

## Constraints

- Do not modify `wormhole-poc/` in any way.
- Do not modify `claude_desktop_config.json` — that is slice 091.
- No `console.log` anywhere in `wormhole/` — stderr only.
- No recursive directory delete tool.
- No read tool — reads stay on VirtioFS.
- Security rules are mandatory, not optional.

## Success criteria

1. `wormhole/server.js`, `config.js`, `logging.js`, `security.js`, `package.json` all exist.
2. Integration tests pass: `node wormhole/tests/security.test.js` exits 0.
3. Path traversal, symlink escape, and absolute path attempts all return errors (tested).
4. `wormhole_append_jsonl` rejects invalid JSON without writing (tested).
5. No `console.log` calls in any `wormhole/` file — stderr only.
6. After every successful tool call, `bridge/wormhole-heartbeat.json` is overwritten with `{ ts, tool, path }`.
7. `bridge/wormhole-heartbeat.json` in `.gitignore`.
8. Committed on `slice/089-wormhole-core`.
