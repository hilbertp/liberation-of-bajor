#055 — Error boundary for relay

Goal

Wrap all relay bridge calls in a try/catch boundary. Surface caught errors to the dashboard health indicator instead of silently swallowing them.

Scope

  • Add error boundary wrapper in bridge/relay.js
  • Emit error events to the watcher heartbeat channel
  • Dashboard picks up errors via existing health-poll endpoint

Out of Scope

Retry logic (covered by #056). Dashboard error UI redesign (separate slice).

Acceptance Criteria

  • ✓ Relay call failure does not crash the watcher process
  • ✓ Error surfaces in dashboard health pill within one poll cycle (5s)
  • ✓ Error message includes relay endpoint + HTTP status code
Rendered tab shows Kira's spec formatted for fast scanning — goal, scope, ACs. This is the default view because most of the time you're reading to approve, not editing. Three actions: Approve sends to O'Brien. Refine sends back to Kira for another pass. Reject drops it entirely.

#053 — Slice detail modal
Sprint 2  ·  O'Brien  ·  done  ·  3m 12s  ·  5.7k tokens  ·  $0.42

Objective

Build a full-screen overlay triggered by clicking Edit on any queue row. Shows Kira's slice spec in two formats: Rendered (default, formatted for scanning) and Source (raw markdown, editable inline). Approval actions are scoped to the item's current queue state.

Context

See ops-dashboard-spec.md — Slice Detail Overlay section. Reference wireframes: ops-ux-concept.html Screens ② and ③. Overlay replaces viewport content below the header — not a modal dialog.

Tasks

  1. Create overlay container with two tabs: Rendered and Source
  2. Rendered tab: parse and display spec sections (Goal, Scope, Out of Scope, Acceptance Criteria)
  3. Source tab: editable textarea, content bound to slice file on disk
  4. Actions footer — staged context: Approve / Refine / Reject; accepted context: Save edits / Send to Kira / Remove from queue
  5. Dismiss on ✕ close or Escape key; no unsaved-changes prompt

Constraints

No external markdown renderer. Parse only the four expected section headers. Do not carry state between open/close cycles — always reads fresh from disk on open.

Acceptance Criteria

  • ✓ Edit on any queue row opens the overlay
  • ✓ Rendered tab shows all four spec sections in correct order
  • ✓ Source tab shows raw markdown in an editable field
  • ✓ Approve moves slice to pending and closes overlay
  • ✓ Reject removes the slice file from disk entirely
  • ✓ Source edits persist to the slice file on disk within 500ms
Read-only view of the original brief sent to O'Brien. Context bar shows sprint, builder, outcome, duration, and cost at a glance. No approval actions — this slice is settled. Opened via Details › on any history row. Closed with ✕ or Escape.
